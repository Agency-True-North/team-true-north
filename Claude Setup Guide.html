01 / How This Works

Claude isn't one thing. There are several versions of it, and most people are using the one that forgets them the moment the tab closes. This guide sets you up with the version that remembers everything.

Here's the difference between the options:

Not Recommended Claude Chat (claude.ai)

The browser-based chat you've probably already used. Every new conversation starts blank. It doesn't know you, your business, or your voice unless you re-explain everything each time. Great for quick one-offs. Not suitable for building a business system.

Not Recommended Claude Cowork (Desktop App)

The desktop app version. Better than browser-based, but still limited in how deeply it can read and manage your files. If you're already using this with files, you can migrate easily — Section 04 tells you how.

Recommended Claude Code in VS Code

This is what this guide sets up. VS Code is a free workspace editor made by Microsoft — think of it as a smart filing cabinet where Claude lives alongside your documents. Claude can read your files, create new ones, edit existing ones, and run tasks for you, all while you watch and approve. It reads your foundation files at the start of every session, so it always knows who you are. This is the setup that compounds.

What "foundation files" means

Foundation files are four documents you build once and keep updated. They teach Claude your voice, your audience, your business goals, and the writing patterns to avoid. Every session Claude reads them first — before it does anything else. That's what makes every output feel like you instead of a generic AI.

02 / Account Setup

Four things to get in place before you build anything. Budget 20 minutes for this section.

1

Create a Claude Account

Go to claude.ai, sign up using your Google email, and upgrade to Claude Pro. Pro is $20/month and required to use Claude Code. Without it, you'll hit usage limits within a few minutes.

2

Download VS Code

VS Code is a free editor made by Microsoft. It's the workspace where Claude will live alongside your files. Download it at code.visualstudio.com/download and follow the instructions for your computer below.

   Mac
  1. Go to code.visualstudio.com/download
  2. Click the .dmg — Apple Silicon button (or Intel if your Mac is older than 2020)
  3. Once downloaded, open the .dmg file from your Downloads folder
  4. Drag the VS Code icon into your Applications folder
  5. Open VS Code from your Applications folder or Launchpad
  6. If you see a security warning, go to System Settings → Privacy & Security and click Open Anyway
⊞   Windows (PC)
  1. Go to code.visualstudio.com/download
  2. Click the Windows — User Installer — x64 button
  3. Run the downloaded .exe file
  4. Accept the license agreement and click Next through the setup
  5. Important: On the "Additional Tasks" screen, check Add to PATH — this is required for Claude Code to work correctly
  6. Also check Add "Open with Code" action for easy folder access
  7. Click Install, then Finish
  8. Open VS Code from your Start menu
Windows users — also install Git

Git is a tool Claude Code uses to manage files. Windows doesn't include it by default. Go to git-scm.com/download/win, download the installer, and run it with all the default settings. Restart VS Code after installation.

3

Install the Claude Code Extension

Inside VS Code:

  • Click the Extensions icon in the left sidebar (it looks like four small squares)
  • Search for Claude Code
  • Find the one by Anthropic and click Install
  • Once installed, a Claude icon will appear in your left sidebar
  • Click it and sign in with the same account you used on claude.ai
4

Create a GitHub Account

GitHub is a free service that backs up your workspace files so they're never lost. Go to github.com and create a free account using the same email you used for Claude. You'll use this in Section 12 — skip it for now and come back once your foundation is built.

Optional: Download the Claude mobile app

The Claude mobile app (iOS and Android) is useful for on-the-go conversations. Once you set up GitHub sync in Section 12, the app can reference your workspace files. For now, all the setup happens on your computer.

03 / Your Folder System

Before Claude can read your files, you need to create the folder where they'll live. This takes five minutes and you only do it once.

1

Create your main Claude folder

On your computer, create a new folder called Claude. Put it somewhere you'll remember — your Desktop or your Documents folder both work. This folder is your workspace. Everything Claude creates and reads will live here.

2

Create two subfolders inside it

Inside your Claude folder, create two folders:

  • ABOUT ME — where your four foundation files will live
  • PROJECTS — where project-specific folders and files will be saved
3

Open the folder in VS Code

In VS Code, go to File → Open Folder and select your Claude folder. VS Code will now show your folder structure in the left panel. This left panel is called the File Tree — it's where all your files and folders are listed.

The Claude icon in the left sidebar is where you'll chat. The main editing area on the right is where files open when you click them.

Keep this folder open every time you work

VS Code remembers your last opened folder, so it usually opens in the right place automatically. If Claude ever says it can't find a file, it's likely because the wrong folder is open. Go to File → Open Folder and select your Claude folder again to fix it.

04 / Where Are You Coming From?

Depending on how you've been using AI, you may already have useful information stored somewhere. This section helps you bring it into your new workspace so you're not starting from scratch.

Starting Fresh New to AI tools

No migration needed. Jump straight to Section 05 and start building your foundation files from scratch.

Migrating Coming from ChatGPT

Your context is stored in ChatGPT — and you can extract it before you leave. Copy all four prompts below and paste them into ChatGPT at once. Save the responses into a single document on your computer.

Copy and paste all four into ChatGPT1. "Look back across our conversations and tell me what you've learned about me as a person and as a business owner." 2. "Summarize what you know about my business." 3. "Summarize what you know about my ideal client." 4. "Pull out writing samples you've created for me that I responded to most positively."

Once you have all four responses saved, paste them into Claude Code when it begins the interview in Section 05. Then tell Claude:

Paste this into Claude Code to start the interviewHere's context from my time using ChatGPT. Use this to draft my foundation files. Only ask me about gaps you can't fill from this material.
Migrating Coming from Claude Chat or the Claude desktop app

Claude Chat and the desktop app don't retain memory between conversations, so there's no stored context to extract. Start the interview in Section 05 from scratch. If you remember specific pieces of work Claude helped with, mention them during the interview — they can be used as voice examples to build from.

Migrating Coming from Claude Cowork with existing files

If you already have foundation files saved in Cowork, just copy them directly into your ABOUT ME folder inside your Claude workspace. You're already most of the way there — skip ahead to Section 06.

05 / Your Four Core Files

These four files are the foundation of your entire Claude setup. Build them once. Update them as your business evolves. Every session Claude reads them before doing anything else.

Each file is built through a guided conversation with Claude. The process takes 30 to 45 minutes and works best when you treat it like an interview — answer thoroughly rather than rushing through it. The more honest and specific you are, the more useful every future output will be.

Before you start

Open a fresh conversation in VS Code. Make sure Claude Code is open in the sidebar. At the bottom of the chat panel, you can select your model — use Claude Opus for this foundational work. It's the most capable model in the Claude family and worth using when you're building files that will shape every future session. (Claude Sonnet is faster and works well for most day-to-day tasks once your foundation is in place.) You'll build all four files in this same session.

Talk instead of type — Wispr Flow

Wispr Flow is a voice-to-text tool that works across every app, including VS Code. It types in real time as you speak — no copy-pasting, no switching windows. These interviews go significantly faster when you can just talk through your answers. Download it at wisprflow.ai, set it up in five minutes, and use it for the entire build process in this section.

File 1: about-me.md

Target length: under 2,000 words. Lives in your ABOUT ME folder.

This file is your voice and your identity. It tells Claude who you are, how you work, what good work looks like to you, and what you absolutely won't tolerate. Every piece of writing Claude produces is filtered through this file.

Paste this prompt into a fresh Claude session to build it:

Paste this prompt into ClaudeYou are going to help me build my about-me.md file for my ABOUT ME folder. This file will be read at the start of every session. Keep it clear, specific, and lean. Under 2,000 words. Interview me, one question at a time. Cover: WHO I AM - What I do, my role, my industry - Who I serve and the relationship I have with them - What a meaningful week of work looks like HOW I WORK - Tools I use day to day - How I move a task from idea to done - My review and editing standards - What "finished" looks like for me WHAT GOOD LOOKS LIKE - An example of recent work I'm proud of and why - What separates strong work from average work in my field WHAT I AVOID - Patterns or shortcuts in my industry that bother me - When AI writes for me and it's wrong, what's usually off MY RULES - Hard lines I won't cross - The two or three things every piece of my work must have Push back gently if my answers are vague. Ask for examples. If I have existing files or context, use them and only ask about gaps. Once done, write the about-me.md file. Don't save the raw Q&A. Extract the patterns. End with a section called "Instructions for Claude" with 8 to 10 clear, actionable rules. Create the file as about-me.md in my ABOUT ME folder.

File 2: ideal-client.md

Target length: under 1,500 words. Lives in your ABOUT ME folder.

This file tells Claude exactly who you serve — the language they use, what they're afraid of, what they want, and where they are in their awareness of the problem you solve. Without this file, Claude writes for a generic audience. With it, every output speaks directly to your person.

Paste this prompt into Claude (same session)Now help me build my ideal-client.md file. This describes the one person my business is built to serve, in enough detail that Claude can write to them like a friend who knows them well. Interview me, one question at a time. Cover: WHO THEY ARE - Demographics: age, life stage, profession, income - Family or relationship situation - A snapshot of their daily life WHAT THEY WANT - The bigger life or feeling they're after - The version of themselves they're becoming - What "winning" looks like to them WHAT THEY STRUGGLE WITH - Their core pain points, in their own words - What keeps them up at night - What they've tried that hasn't worked WHAT THEY FEAR - The deeper fears under the surface frustration - What they don't want their life to look like in 5 years THEIR LANGUAGE - Phrases that resonate, the ones that make them say "this is for me" - Phrases to avoid because they feel like marketing THEIR AWARENESS - How aware they are of the problem and of solutions If I have a profile or avatar built, ask me to share it and use it as the starting point. Make the file scannable: short sections, real language, no fluff. Create the file as ideal-client.md in my ABOUT ME folder.

File 3: my-company.md

Target length: under 1,000 words. Lives in your ABOUT ME folder.

This file captures what your business is focused on right now — your goals, your current priorities, and what you're deliberately saying no to. It gives Claude the strategic context so its suggestions stay relevant and don't pull you off course.

Paste this prompt into Claude (same session)Now help me build my my-company.md file. About-me.md covers who I am. Ideal-client.md covers who I serve. This file is only about my business goals, current focus, and decisions. No overlap. Interview me, one question at a time, 6 to 8 questions: GOALS - My top 2 to 3 goals for this year, with specific numbers or milestones - The one metric that would make this year a success - Revenue, audience, or product targets FOCUS - What I'm spending most of my energy on this quarter - Platforms or channels that matter most right now DECISIONS - What I'm actively saying no to - What I recently stopped doing, and why - Anything I'm betting on that most people in my field aren't Output: short, scannable. Bullet points over paragraphs. Three sections: Goals, Focus right now, Saying no to. Under 1,000 words. Create the file as my-company.md in my ABOUT ME folder.

File 4: anti-ai-writing-style.md

Target length: under 800 words. Lives in your ABOUT ME folder.

This is your calibration file. It's a list of everything that makes AI writing sound like AI writing — the filler words, the hollow phrases, the corporate formatting, the patterns that make people stop reading. The more specific you make this file, the harder Claude will work to sound like a real person.

Paste this prompt into Claude (same session)Last file. Help me build my anti-ai-writing-style.md. Interview me, one question at a time. Cover: WORDS I DON'T USE - Common AI words I find hollow (delve, harness, tapestry, leverage, unlock). Ask which ones I want to ban. - Industry jargon I avoid - Marketing cliches that bother me PATTERNS I REJECT - The "this isn't X, this is Y" reframe pattern - Excessive sentence fragments - Overuse of bullet points where prose would work FORMATTING I CARE ABOUT - Preferred sentence length - Preferred paragraph length - Capitalization preferences - Informal vs formal punctuation WHAT MY WRITING SHOULD ALWAYS DO - The 2 to 3 things every piece should achieve - Voice qualities that have to be present Write the file as a clear list of rules under section headings. Make it actionable. Create the file as anti-ai-writing-style.md in my ABOUT ME folder.
Update these files as your business evolves

Your business will change. Your goals will shift. New things will belong in these files, and old things will become outdated. Review all four files monthly and update them. Any change you make applies immediately to the next conversation — Claude reads them fresh every time.

06 / Your CLAUDE.md File

CLAUDE.md is the master instruction file. It tells Claude what to read, what to ignore, and how to behave in every single session. It lives at the root of your Claude folder — not inside any subfolder.

Create a new file called CLAUDE.md in your main Claude folder (not inside ABOUT ME or PROJECTS — directly in Claude). Then paste the following into it:

Your CLAUDE.md file content# [Your Name]'s Workspace Before any task, read every file in the About Me folder: - about-me.md — who I am, how I work, what I value - ideal-client.md — who I serve, the language they use - my-company.md — my current goals, focus, what I'm saying no to - anti-ai-writing-style.md — words, patterns, and formatting to avoid Use these together. About-me sets my voice. Ideal-client sets my audience. My-company sets the strategic context. Anti-ai-writing-style is the calibration layer. Do not read the Projects folder on your own. Only read it when I point you to a specific file or subfolder. For active workstreams, I'll tell you which project we're working on. Read the files inside that project folder. Save deliverables back into that same project folder. If a brief is unclear, ask me. Don't fill gaps with guesses. Don't pad responses. Deliver clean work.
Why CLAUDE.md matters

This file is what makes your setup automatic. Without it, you'd need to re-explain your context at the start of every session. With it, Claude reads your foundation files before it does anything else — every time. Edit it whenever your workflow changes. The changes take effect immediately in your next conversation.

07 / Verify and Daily Workflow

Before you start using Claude for real work, verify that the setup is working correctly. Then build the daily habits that make the system compound over time.

Verifying Your Setup

Start a new conversation in VS Code and paste this prompt:

Verification promptRead my CLAUDE.md file and all files in my ABOUT ME folder. Then tell me: 1. What do you know about who I am and how I work? 2. What do you know about who I serve? 3. What are my current business goals and priorities? 4. What writing patterns should you avoid? Flag anything that seems thin, missing, or unclear so I know what to strengthen.

If Claude comes back with accurate, detailed answers — your foundation is working. If it flags gaps or gets something wrong, update the relevant file and run it again.

How to Start a Session

Every new conversation in VS Code starts with Claude reading your CLAUDE.md, which triggers it to read your foundation files. You don't need to paste context every time — just tell Claude what you're working on.

Examples of good session openers:

For a new projectI'm starting a new project called [name]. Create a folder for it inside my PROJECTS folder, then let's begin.
For an existing projectRead my CLAUDE.md and then open the [project name] folder in PROJECTS. Here's what we're working on today: [brief]
For a one-off piece of contentI need [type of content] for [purpose]. It's for [audience description]. Here are the key points to include: [list]
For strategy workI want to think through [topic]. Read my foundation files first, then let's have a working conversation about this — I'll share my thinking and I want pushback, suggestions, and clarity.

How to Close a Session

At the end of any important conversation, use one or more of these prompts before closing:

Save session notesSummarize what we decided and built today. Save a session-notes.md file in the [project name] folder so we have a record.
Draft the next openerWrite an opening prompt I can paste at the start of our next session to pick up exactly where we left off.

Weekly Maintenance

Your foundation files are only as useful as they are current. Set a recurring reminder to review them every week — 10 minutes is enough. To automate this, you can create a weekly check-in skill that reviews your files and flags what needs updating.

To create the skill, paste this prompt into Claude:

Create the weekly check-in skillCreate a custom slash command called weekly-check-in. Save it as a markdown file at .claude/commands/weekly-check-in.md in my Claude workspace. The command should instruct Claude to: 1. Read all four foundation files in my ABOUT ME folder 2. Review each one for content that may be outdated, missing, or too thin 3. Suggest specific edits or additions for each file, quoting the exact text to change 4. Flag any file that has grown beyond its target length (about-me: 2,000 words / ideal-client: 1,500 words / my-company: 1,000 words / anti-ai: 800 words) 5. Ask me one question: what has changed in your business since the last review? After creating the file, confirm the path so I can verify it exists.

After creating it, type /weekly-check-in at the start of any session and Claude will run through all four files and surface what needs attention. If the command doesn't trigger, confirm the file was saved to .claude/commands/weekly-check-in.md — the folder and file name must match exactly.

08 / Quick Reference and Troubleshooting

Common problems and how to fix them. If something feels off, check here first before assuming the setup is broken.

The Problem What's Causing It How to Fix It
Writing sounds generic anti-ai-writing-style.md is too vague Add more specific examples of words and patterns to avoid
Claude keeps asking what it already knows CLAUDE.md isn't being read — likely a file path issue Confirm CLAUDE.md is in the root Claude folder, not inside a subfolder
Decisions or goals aren't reflected my-company.md is out of date Update it to reflect your current focus and priorities
File not found errors Wrong folder is open in VS Code File → Open Folder → select your Claude folder
Responses getting slow or confused Conversation has gotten too long Start a fresh conversation — long threads degrade quality over time
Output doesn't match your audience ideal-client.md needs more detail Add specific language your clients use and update the awareness level
Claude goes off-brief without asking Instruction was ambiguous Be more specific in your prompt, or add "ask me if anything is unclear" at the end
Reminders Keep these habits in place
  • One task per conversation. Long, multi-topic threads produce worse output. Start fresh when you switch focus.
  • CLAUDE.md runs every session. Any edits you make take effect immediately in the next conversation.
  • Update foundation files monthly at minimum. Stale files produce stale output.
  • You are always in authority. Claude proposes. You decide. That's the relationship.

09 / Bring In Existing Assets (Optional)

If you have existing content — past campaigns, brand guidelines, scripts, email sequences — Claude can audit them and pull out what's worth keeping in your system.

Drop relevant files into a BRAND folder inside your Claude workspace, then run this prompt:

Asset audit promptI've added some existing brand assets and content to my BRAND folder. Please review everything in there and tell me: 1. What content is still relevant and worth keeping? 2. What patterns do you notice in my existing work — voice, structure, recurring themes? 3. What should be added to my foundation files based on what you found? 4. Is there anything I should retire or stop doing?

To set up a dedicated BRAND folder that Claude references before any visual or design work:

Brand folder setup promptCreate a BRAND folder in my Claude workspace. Inside it, create a brand-basics.md file that documents: - My brand colors (I'll provide the hex codes) - My fonts - Logo usage rules - Voice and tone notes - Things that are off-brand Then update my CLAUDE.md to tell Claude to read brand-basics.md before any visual or design-related work.

10 / Secure Your API Keys (Optional)

If you start using Claude to run automations or connect to other tools, you'll be working with API keys — private credentials that should never appear in your chat history or files. Doppler is the tool that keeps them safe.

Doppler is a free secrets vault. Instead of pasting credentials into chat or saving them in your files, Doppler injects them securely at runtime — meaning Claude can use them without them ever being exposed.

Critical rule: never paste API keys into chat

If an API key appears in a conversation, in a file, or anywhere Claude can see it in plain text, it is no longer secure. Rotate it immediately and set up Doppler before continuing.

1

Create a free Doppler account

Go to doppler.com and sign up for free. No credit card required.

2

Ask Claude to install the Doppler CLI

In your VS Code chat, paste: "Install the Doppler CLI on my computer so I can use it to manage API keys securely." Claude will walk you through the installation steps for your operating system.

3

Create a project in Doppler and add your keys

In your Doppler dashboard, create a project. Add your API keys as environment variables with clear names in all-caps — for example: OPENAI_API_KEY, GHL_API_KEY. Never name them vaguely.

4

Run tools through Doppler

When running scripts that need API keys, use this format: doppler run -- python3 script.py. Doppler injects the keys at runtime without exposing them in the command or in any file.

Once Doppler is set up, add this line to your CLAUDE.md:

Add to CLAUDE.mdSecurity rule: Never paste API keys, passwords, or credentials into the chat. If an automation requires a credential, retrieve it from Doppler using `doppler run --` and confirm with me before proceeding.

11 / Access Your Foundation Files on Mobile (Optional)

Your foundation files live on your computer. But there are two ways to bring that same context into mobile Claude conversations so you're not starting from scratch on your phone.

Option A — Claude Projects (Simplest)

Claude Projects is a feature inside claude.ai that lets you upload files Claude will reference in every conversation within that project. It works on both desktop and mobile.

1

Create a Project on claude.ai

Go to claude.ai, click Projects in the left sidebar, and create a new project called My Foundation.

2

Upload your four foundation files

Inside the project, upload about-me.md, ideal-client.md, my-company.md, and anti-ai-writing-style.md. These will be attached to every conversation in the project.

3

Use the project on mobile

Download the Claude app (iOS or Android). Sign in with the same account. Open the project — Claude will have access to your foundation files in every conversation inside it.

4

Keep files current

When you update a foundation file in VS Code, re-upload it to your Claude Project. Delete the old version and upload the new one. It takes under a minute.

Option B — GitHub Backup (For Archiving and Safety)

Even if you don't need mobile access, backing your Claude workspace up to a private GitHub repository is a good habit. If you ever lose your computer or need to set up on a new machine, your entire foundation is recoverable in minutes.

Run this before your first push

Before syncing to GitHub, ask Claude to check your workspace for sensitive content: "Review all files in my Claude workspace and flag anything sensitive — passwords, personal details, private client info, or anything I'd be uncomfortable having stored online. Create a .gitignore file for anything that should be excluded."

1

Connect your workspace to a private GitHub repo

In your VS Code chat, paste: "Connect my local Claude folder to a new private GitHub repository and push the contents. Walk me through each step." Claude will guide you through creating the repo and completing the first push.

2

Keep it updated

After any significant update to your foundation files, push the changes. Open the terminal in VS Code and run: git add . && git commit -m "update files" && git push. Or just ask Claude to push for you.

12 / What's Next — Building Automations

Once your foundation is solid, you can start building automations — recurring tasks that Claude handles for you with minimal input. Campaign creation, data population, report formatting, content scheduling. The same Plan Mode approval process applies.

The key to good automations is front-loading the context. A well-briefed Claude that understands your system fully will handle 90% of a task correctly. The remaining 10% — review, judgment calls, final approval — stays with you. That's not a limitation. That's the right division of labour.

Train Claude to be resourceful

The most common automation frustration is Claude stopping to ask questions you've already answered in your foundation files. If this happens, your files probably need updating. The more complete your context, the less Claude needs to interrupt you — and the more it can handle independently.

To kick off any new automation project, paste this prompt:

Automation kickoff promptI want to build an automation for [describe the task]. Before you build anything: 1. Read my CLAUDE.md and foundation files 2. Create a new folder inside PROJECTS called [automation name] 3. Draft a plan: what the automation will do, what inputs it needs from me, what it will produce, and where it will save the output 4. Present the plan for my review before writing a single line I'll approve the plan before we proceed.

For deeper learning on automation and what's possible with Claude, the claude101.com resource by Ruben Hassid provides structured guides for building more advanced workflows.
